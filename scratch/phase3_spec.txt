Implementation Rules
Add the following implementation constraints for Phase 3:
1. Dependency
If PyMuPDF is used, add dependency in pyproject.toml as: pymupdf>=1.24.0.
Code may import it as: import fitz.
Use PyMuPDF only to rasterize PDF pages into images. Do not use it to extract text, tables, blocks, words, or quotation content.
2. FilePathsModel
If FilePathsModel.page_manifest is added, it must have this default: page_manifest: str = "source/page_manifest.json".
This keeps Phase 1 and Phase 2 packages backward compatible.
3. Manifest path convention
All paths in page_manifest.json must be relative to package root.
Examples: source/original.pdf and source/pages/page_0001.png. Do not write absolute paths such as D:\...
4. Manifest validation
After writing source/page_manifest.json, validate it with PageManifestModel.
If a new schema is added, generate schemas/page_manifest.schema.json and update scripts/generate_schemas.py.
5. Atomic / overwrite behavior
If overwrite=False and any output image or source/page_manifest.json already exists, fail clearly before rasterization starts.
Do not create partial output and then fail halfway.
If overwrite=True, rewriting all page images and the manifest is allowed.
6. Supported image format
Phase 3 only needs to support png.
If CLI accepts --format, currently accept only png. Any other format must fail clearly.
Do not add jpeg/webp unless explicitly requested.
7. DPI validation
dpi must be a positive integer. If dpi <= 0, fail clearly. Default is 150.
8. Encrypted PDF
If source/metadata.json has encrypted: true, prepare-pages must fail before rasterization.
Do not decrypt. Do not ask for a password. Do not continue processing.
9. Audit logging
pdf_page_rasterized may be logged once after all pages are rasterized, with details: page_count, dpi, image_format, output_dir.
Do not log one event per page; avoid overly long audit logs.
10. Scope reminder
Phase 3 only creates page images and page_manifest.
Do not generate parsed/quotation.json, parsed/quotation.md, normalized.json from PDF, OCR text, extracted text, or table data.
